Guiding principles

Keep the writing journey learner-owned while surfacing the metrics teachers need to personalize feedback. WriterCoach orchestrates the supporting workflow without ever brokering paid AI access.

LLM-agnostic

Students bring transcripts from any free or local LLM interface, so classrooms stay adaptable as tools change.

Costless by design

Assemble the platform with open-source services and free tiers to keep recurring fees at zero.

Learner owned

Drafts stay on the student’s device until they choose to upload a submission package.

Transparency

Teachers see revision history, feedback artifacts, and student reflections in a single view.

Student experience

Five simple steps keep learners focused on reflection and craft.

  1. Prompt intake

    Access the weekly brief and borrow templates for asking any LLM clarifying questions.

  2. Local drafting

    Compose in an offline editor while retaining the source files on personal devices.

  3. Feedback loop

    Consult the LLM for grammar or style suggestions using guided prompt templates.

  4. Submission package

    Upload the final draft, the LLM transcript, and a reflection checklist via a structured form.

  5. Progress snapshot

    View dashboards summarizing milestones, word counts, and pending teacher comments.

Architecture overview

Free-tier friendly stack choices keep deployments lightweight and portable.

Frontend

SvelteKit static export + Tailwind for responsive dashboards.

Hosting

GitHub Pages with optional Cloudflare Pages fallback.

Auth & Database

Supabase free tier with row-level security policies.

File storage

Supabase buckets or Cloudflare R2 for uploads and transcripts.

Automation

n8n community edition or GitHub Actions for workflows.

Local utilities

Student scripts to format transcripts and compute word counts.

Minimal data model

Tables focus on metadata and reflections—drafts remain student controlled.

Table Key fields Notes
students id, display_name, cefr_level, preferred_tools, guardian_contact Support pseudonyms and optional guardian links.
assignments id, title, instructions, due_date, llm_prompt_templates Provide copy-ready prompt suggestions.
submissions id, student_id, assignment_id, submitted_at, word_count, transcript_url, draft_url, self_reflection Store artifact links with lightweight metadata.
feedback id, submission_id, author_role, comment_text, tags, created_at Enable teacher and peer discussion threads.
metrics_daily student_id, date, minutes_spent, words_written, llm_used Optional streak tracking for coaching conversations.
